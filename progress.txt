## 2026-04-01 — US-003: Approccio A — Event-based disaggregation

Implemented scripts/nilm/approach_event_based.py.

- `run(signal, devices, threshold_w=50.0)`: standard interface returning dict[str, pd.Series]
- Computes delta_w = signal.diff(); detects ON events (delta_w > threshold) and OFF events
  (delta_w < -threshold)
- For each ON event: selects device with |p_typical - |delta_w|| minimum among present devices
  (prior_weight >= 1.0); falls back to all devices if none are present
- Pairs each ON event with the first OFF event within dur_typical*3 minutes; uses dur_typical
  as default duration if no OFF event is found
- Output: per-device pd.Series with 0 (OFF) or p_typical_w (ON); NaN where signal is NaN

## 2026-04-01 — US-004: Approccio B — HMM per singolo dispositivo (unsupervised)

Implemented scripts/nilm/approach_hmm.py and added hmmlearn>=0.3 to requirements.txt.

- `run(signal, devices)`: standard interface returning dict[str, pd.Series]
- For each present device (prior_weight >= 1.0):
  - Fits GaussianHMM(n_components=2, covariance_type='diag', n_iter=50, random_state=42)
  - Initializes means_ = [[5.0], [p_typical_w]], covars_ = [[100], [(p_typical_w*0.2)^2]]
  - Fits on the residual signal (aggregate minus already-estimated devices), NaN dropped
  - Predicts state sequence; maps higher-mean state index to ON
  - Assigns p_typical_w for ON states, 0 for OFF; NaN where signal is NaN
- Skips devices whose residual is entirely NaN (leaves as zero series)

## 2026-04-01 — US-005: Approccio C — Factorial HMM semplificato

Implemented scripts/nilm/approach_fhmm.py.

- `run(signal, devices, max_iter=50, tol=1e-3)`: standard interface returning dict[str, pd.Series]
- Greedy coordinate-ascent FHMM:
  - Initializes all device binary states to 0 for every timestep
  - At each iteration, for each device i: computes residual = signal - sum(other devices);
    sets x_i(t)=1 if residual(t) > p_typical_i/2 (i.e., ON state minimises |residual - p_i*x_i|)
  - Repeats until total state changes < tol*T or max_iter reached
- Temporal smoothing: removes ON blocks shorter than dur_min_min/2 samples after convergence
- NaN positions in signal are treated as 0 for inference; output NaN-masked to match signal
- Only operates on present devices (prior_weight >= 1.0)

## 2026-04-01 — US-006: Approccio D — Template matching

Implemented scripts/nilm/approach_template.py.

- `run(signal, devices, corr_threshold=0.6)`: standard interface returning dict[str, pd.Series]
- For each present device (prior_weight >= 1.0) with dur_typical_min >= 3:
  - Builds rectangular numpy template of length int(dur_typical_min) at value p_typical_w
  - Computes normalized cross-correlation via scipy.signal.correlate(mode='same') divided by
    template L2 norm
  - Detects peaks above corr_threshold with distance=int(dur_typical_min*0.5) using find_peaks
  - Assigns ON windows greedy (sorted by descending correlation score); skips windows that
    overlap already-assigned timesteps (first-come-first-served)
- NaN positions in signal are treated as 0 for correlation; output NaN-masked to match signal
- Devices processed in descending p_typical_w order (larger appliances first)

## 2026-04-01 — US-007: Approccio E — Event-based con prior pesato da inventario

Implemented scripts/nilm/approach_event_prior.py.

- `run(signal, devices, threshold_w=50.0, sigma_factor=0.3)`: standard interface returning dict[str, pd.Series]
- Event detection: same delta_w threshold method as US-003 (ON events: delta_w > threshold,
  OFF events: delta_w < -threshold)
- Device assignment via Bayesian score for each event:
  - likelihood = N(|delta_w|; p_typical_w, (sigma_factor * p_typical_w)^2) — Gaussian centered at
    the device's typical power
  - frequency_factor = min(frequency_per_week / 7.0, 1.0)
  - score = likelihood * prior_weight * frequency_factor
  - All devices considered (present prior_weight=1.0 and absent prior_weight=0.05)
- ON/OFF pairing: same logic as US-003 (first OFF within dur_typical*3 min; fallback to
  dur_typical_min duration)
- Output: per-device pd.Series with 0 (OFF) or p_typical_w (ON); NaN where signal is NaN

- `compute_metrics(signal, disaggregation, devices)`: computes proxy quality metrics:

  - Iterates all (imei, approach) pairs in results dict

  - Saves analysis/benchmark_summary.png as a heatmap (matplotlib imshow) of mean metrics
    per approach across all IMEIs, with numeric annotations
